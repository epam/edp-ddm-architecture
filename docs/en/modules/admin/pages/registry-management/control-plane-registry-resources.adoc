Istio sidecar ::

//*Sidecar* -- —Ü–µ –¥–æ–¥–∞—Ç–∫–æ–≤–∏–π –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä, —è–∫–∏–π –∑–∞–ø—É—Å–∫–∞—î—Ç—å—Å—è –ø–æ—Ä—è–¥ –∑ –æ—Å–Ω–æ–≤–Ω–∏–º –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–æ–º —É –ø–æ–¥—ñ OpenShift. *Istio* –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É—î –ø—ñ–¥—Ö—ñ–¥ *sidecar* –¥–ª—è –≤–Ω–µ—Å–µ–Ω–Ω—è –∑–º—ñ–Ω —É –º–µ—Ä–µ–∂–µ–≤—ñ –Ω–∞–ª–∞—à—Ç—É–≤–∞–Ω–Ω—è –±–µ–∑ –Ω–µ–æ–±—Ö—ñ–¥–Ω–æ—Å—Ç—ñ –∑–º—ñ–Ω–∏ —Å–∞–º–æ–≥–æ –¥–æ–¥–∞—Ç–∫—É.
*Sidecar* is an additional container that runs alongside the main container in an OpenShift pod. *Istio* uses the sidecar approach to make changes to the network configuration without modifying the application itself.
+
//* –ê–∫—Ç–∏–≤—É–π—Ç–µ –ø–∞—Ä–∞–º–µ—Ç—Ä *Enabled*. +
* Activate the *Enabled* parameter.
//–¶–µ–π –ø–∞—Ä–∞–º–µ—Ç—Ä –≤–∫–∞–∑—É—î, —á–∏ –≤–∫–ª—é—á–µ–Ω–æ –≤–∏–∫–æ—Ä–∏—Å—Ç–∞–Ω–Ω—è sidecar Istio –¥–ª—è —Ü—å–æ–≥–æ –∫–æ–Ω–∫—Ä–µ—Ç–Ω–æ–≥–æ —Å–µ—Ä–≤—ñ—Å—É.
This parameter indicates whether Istio sidecar is enabled for this specific service.
+
//* –ù–∞–ª–∞—à—Ç—É–π—Ç–µ –ø–∞—Ä–∞–º–µ—Ç—Ä–∏ *Requests* i *Limits*.
* Configure the *Requests* and *Limits* parameters.
+
//–¶—ñ –ø–∞—Ä–∞–º–µ—Ç—Ä–∏ –≤–∫–∞–∑—É—é—Ç—å –Ω–∞ –æ–ø—Ç–∏–º–∞–ª—å–Ω—ñ (*Requests*) —Ç–∞ –º–∞–∫—Å–∏–º–∞–ª—å–Ω—ñ (*Limits*) —Ä–µ—Å—É—Ä—Å–∏, —è–∫—ñ –º–∞—é—Ç—å –±—É—Ç–∏ –≤–∏–¥—ñ–ª–µ–Ω—ñ –¥–ª—è Istio sidecar.
These parameters specify the optimal (*Requests*) and maximum (*Limits*) resources to be allocated for the Istio sidecar.
+
//*Requests* -- —Ü–µ –º—ñ–Ω—ñ–º—É–º —Ä–µ—Å—É—Ä—Å—ñ–≤, —è–∫—ñ OpenShift –≥–∞—Ä–∞–Ω—Ç—É—î –¥–ª—è –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞. –£ –Ω–∞—à–æ–º—É –ø—Ä–∏–∫–ª–∞–¥—ñ -- —Ü–µ `350m` CPU —ñ `128Mi` –ø–∞–º'—è—Ç—ñ –¥–ª—è Istio sidecar. –Ø–∫—â–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä –ø–æ—Ç—Ä–µ–±—É—î –±—ñ–ª—å—à–µ —Ä–µ—Å—É—Ä—Å—ñ–≤, —ñ —è–∫—â–æ —Ü—ñ –¥–æ–¥–∞—Ç–∫–æ–≤—ñ —Ä–µ—Å—É—Ä—Å–∏ –¥–æ—Å—Ç—É–ø–Ω—ñ, OpenShift –∑–º–æ–∂–µ —ó—Ö –Ω–∞–¥–∞—Ç–∏.
*Requests* represent the minimum resources guaranteed by OpenShift for the container. In our example, it is 350m CPU and 128Mi memory for the Istio sidecar. If the container requires more resources, and they are available, OpenShift can provide them.
+
//*Limits* -- —Ü–µ –º–∞–∫—Å–∏–º—É–º —Ä–µ—Å—É—Ä—Å—ñ–≤, —è–∫—ñ OpenShift –¥–æ–∑–≤–æ–ª–∏—Ç—å –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä—É –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É–≤–∞—Ç–∏. –£ –Ω–∞—à–æ–º—É –ø—Ä–∏–∫–ª–∞–¥—ñ -- —Ü–µ `350m` CPU, `128Mi` –ø–∞–º'—è—Ç—ñ –¥–ª—è Istio sidecar. –Ø–∫—â–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä —Å–ø—Ä–æ–±—É—î –≤–∏–∫–æ—Ä–∏—Å—Ç–∞—Ç–∏ –±—ñ–ª—å—à–µ —Ä–µ—Å—É—Ä—Å—ñ–≤, –≤—ñ–Ω –º–æ–∂–µ –±—É—Ç–∏ –ø—Ä–∏–º—É—Å–æ–≤–æ –∑—É–ø–∏–Ω–µ–Ω–∏–π –∞–±–æ –ø–µ—Ä–µ–≤–µ–¥–µ–Ω–∏–π –Ω–∞ –Ω–∏–∂—á–∏–π –ø—Ä—ñ–æ—Ä–∏—Ç–µ—Ç —É —á–µ—Ä–∑—ñ —Ä–æ–∑–∫–ª–∞–¥—É —Ä–æ–∑–≥–æ—Ä—Ç–∞–Ω–Ω—è –ø–æ–¥—ñ–≤ –Ω–∞ –Ω–æ–¥–∞—Ö.
*Limits* represent the maximum resources allowed for the container by OpenShift. In our example, it is 350m CPU and 128Mi memory for the Istio sidecar. If the container tries to use more resources, it may be forcibly stopped or demoted to a lower priority in the deployment queue on the nodes.

Container ::

//*Container* -- –æ—Å–Ω–æ–≤–Ω–∏–π –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä —ñ–∑ –¥–æ–¥–∞—Ç–∫–æ–º.
*Container* is the main container with the application.
+
//* –ù–∞–ª–∞—à—Ç—É–π—Ç–µ –ø–∞—Ä–∞–º–µ—Ç—Ä–∏ *Requests* i *Limits*.
* Configure the *Requests* and *Limits* parameters.
+
//–¶—ñ –ø–∞—Ä–∞–º–µ—Ç—Ä–∏ –≤–∫–∞–∑—É—é—Ç—å –Ω–∞ –æ–ø—Ç–∏–º–∞–ª—å–Ω—ñ (*Requests*) —Ç–∞ –º–∞–∫—Å–∏–º–∞–ª—å–Ω—ñ (*Limits*) —Ä–µ—Å—É—Ä—Å–∏, —è–∫—ñ –º–∞—é—Ç—å –±—É—Ç–∏ –≤–∏–¥—ñ–ª–µ–Ω—ñ –¥–ª—è –æ—Å–Ω–æ–≤–Ω–æ–≥–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞.
These parameters specify the optimal (*Requests*) and maximum (*Limits*) resources to be allocated for the main container.
+
//*Requests* -- —Ü–µ –º—ñ–Ω—ñ–º—É–º —Ä–µ—Å—É—Ä—Å—ñ–≤, —è–∫—ñ OpenShift –≥–∞—Ä–∞–Ω—Ç—É—î –¥–ª—è –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞. –£ –Ω–∞—à–æ–º—É –ø—Ä–∏–∫–ª–∞–¥—ñ -- —Ü–µ `1` CPU, `2Gi` –ø–∞–º'—è—Ç—ñ –¥–ª—è –æ—Å–Ω–æ–≤–Ω–æ–≥–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞. –Ø–∫—â–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä –ø–æ—Ç—Ä–µ–±—É—î –±—ñ–ª—å—à–µ —Ä–µ—Å—É—Ä—Å—ñ–≤, —ñ —è–∫—â–æ —Ü—ñ –¥–æ–¥–∞—Ç–∫–æ–≤—ñ —Ä–µ—Å—É—Ä—Å–∏ –¥–æ—Å—Ç—É–ø–Ω—ñ, OpenShift –º–æ–∂–µ —ó—Ö –Ω–∞–¥–∞—Ç–∏.
*Requests* represent the minimum resources guaranteed by OpenShift for the container. In our example, it is `1` CPU and `2Gi` memory for the main container. If the container requires more resources, and they are available, OpenShift can provide them.
+
//*Limits* -- —Ü–µ –º–∞–∫—Å–∏–º—É–º —Ä–µ—Å—É—Ä—Å—ñ–≤, —è–∫—ñ OpenShift –¥–æ–∑–≤–æ–ª–∏—Ç—å –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä—É –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É–≤–∞—Ç–∏. –£ –Ω–∞—à–æ–º—É –ø—Ä–∏–∫–ª–∞–¥—ñ -- —Ü–µ `1` CPU, `2Gi` –ø–∞–º'—è—Ç—ñ –¥–ª—è –æ—Å–Ω–æ–≤–Ω–æ–≥–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞. –Ø–∫—â–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä —Å–ø—Ä–æ–±—É—î –≤–∏–∫–æ—Ä–∏—Å—Ç–∞—Ç–∏ –±—ñ–ª—å—à–µ —Ä–µ—Å—É—Ä—Å—ñ–≤, –≤—ñ–Ω –º–æ–∂–µ –±—É—Ç–∏ –ø—Ä–∏–º—É—Å–æ–≤–æ –∑—É–ø–∏–Ω–µ–Ω–∏–π –∞–±–æ –ø–µ—Ä–µ–≤–µ–¥–µ–Ω–∏–π –Ω–∞ –Ω–∏–∂—á–∏–π –ø—Ä—ñ–æ—Ä–∏—Ç–µ—Ç —É —á–µ—Ä–∑—ñ —Ä–æ–∑–∫–ª–∞–¥—É —Ä–æ–∑–≥–æ—Ä—Ç–∞–Ω–Ω—è –ø–æ–¥—ñ–≤ –Ω–∞ –Ω–æ–¥–∞—Ö.
*Limits* represent the maximum resources allowed for the container by OpenShift. In our example, it is `1` CPU and `2Gi` memory for the main container. If the container tries to use more resources, it may be forcibly stopped or demoted to a lower priority in the deployment queue on the nodes.
+
//* +++<b style="font-weight: 600">–ó–º—ñ–Ω–Ω—ñ –æ—Ç–æ—á–µ–Ω–Ω—è</b>+++ (–∞–±–æ *environment variables*) -- —Ü–µ –¥–∏–Ω–∞–º—ñ—á–Ω—ñ –Ω–∞–∑–≤–∏ –∑–Ω–∞—á–µ–Ω—å, —â–æ –∑–±–µ—Ä—ñ–≥–∞—é—Ç—å—Å—è –≤ —Å–∏—Å—Ç–µ–º—ñ –π –º–æ–∂—É—Ç—å –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É–≤–∞—Ç–∏—Å—è —Ä—ñ–∑–Ω–∏–º–∏ –ø—Ä–æ–≥—Ä–∞–º–∞–º–∏. –í–æ–Ω–∏ –æ—Å–æ–±–ª–∏–≤–æ –∫–æ—Ä–∏—Å–Ω—ñ –≤ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∏–∑–æ–≤–∞–Ω–∏—Ö —Ç–∞ —Ä–æ–∑–ø–æ–¥—ñ–ª–µ–Ω–∏—Ö —Å–µ—Ä–µ–¥–æ–≤–∏—â–∞—Ö, —Ç–∞–∫–∏—Ö —è–∫ –ü–ª–∞—Ç—Ñ–æ—Ä–º–∞ —Ä–µ—î—Å—Ç—Ä—ñ–≤, –¥–µ –∫–æ–∂–µ–Ω –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä –∞–±–æ –ø–æ–¥ –º–æ–∂–µ –º–∞—Ç–∏ —Å–≤–æ—ó –≤–ª–∞—Å–Ω—ñ –∑–º—ñ–Ω–Ω—ñ –æ—Ç–æ—á–µ–Ω–Ω—è. –¶–µ –¥–∞—î –∑–º–æ–≥—É –∫–µ—Ä—É–≤–∞—Ç–∏ –∫–æ–Ω—Ñ—ñ–≥—É—Ä–∞—Ü—ñ—î—é —Ç–∞ –ø–æ–≤–µ–¥—ñ–Ω–∫–æ—é –∫–æ–∂–Ω–æ–≥–æ –∫–æ–Ω—Ç–µ–π–Ω–µ—Ä–∞ –∞–±–æ –ø–æ–¥–∞ —ñ–Ω–¥–∏–≤—ñ–¥—É–∞–ª—å–Ω–æ.

* *Environment variables*  are dynamic name-value pairs stored in the system and can be used by various programs. They are particularly useful in containerized and distributed environments like the registry platform, where each container or pod can have its own environment variables. This allows individual control over the configuration and behavior of each container or pod.
+
//–ó–º—ñ–Ω–Ω–∞ `JAVA_OPTS` –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É—î—Ç—å—Å—è –¥–ª—è –Ω–∞–ª–∞—à—Ç—É–≤–∞–Ω–Ω—è –ø–∞—Ä–∞–º–µ—Ç—Ä—ñ–≤ JVM (Java Virtual Machine).
The `JAVA_OPTS` variable is used to configure the JVM (Java Virtual Machine) parameters.
+
//–£ —Ü—å–æ–º—É –≤–∏–ø–∞–¥–∫—É, –≤–∫–∞–∑–∞–Ω—ñ –ø–∞—Ä–∞–º–µ—Ç—Ä–∏ `-Xms1536m` —ñ `-Xmx1536m` –≤—Å—Ç–∞–Ω–æ–≤–ª—é—é—Ç—å –º—ñ–Ω—ñ–º–∞–ª—å–Ω–∏–π (`-Xms`) —Ç–∞ –º–∞–∫—Å–∏–º–∞–ª—å–Ω–∏–π (`-Xmx`) —Ä–æ–∑–º—ñ—Ä –ø–∞–º'—è—Ç—ñ, —è–∫–∏–π JVM –º–æ–∂–µ –≤–∏–∫–æ—Ä–∏—Å—Ç–æ–≤—É–≤–∞—Ç–∏.
In this case, the specified parameters `-Xms1536m` and `-Xmx1536m` set the minimum (`-Xms`) and maximum (`-Xmx`) memory size that the JVM can use.
+
//TIP: –í–∏ –º–æ–∂–µ—Ç–µ –ø—Ä–∏–±—Ä–∞—Ç–∏ –∑–º—ñ–Ω–Ω—ñ –æ—Ç–æ—á–µ–Ω–Ω—è –∑ –Ω–∞–ª–∞—à—Ç—É–≤–∞–Ω—å, –Ω–∞—Ç–∏—Å–Ω—É–≤—à–∏ –Ω–∞ –∫–Ω–æ–ø–∫—É *`-`*.
