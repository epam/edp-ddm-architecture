== General overview

//_–ê–¥–º—ñ–Ω—ñ—Å—Ç—Ä–∞—Ç–∏–≤–Ω–∞ –∑–æ–Ω–∞ –ü–ª–∞—Ç—Ñ–æ—Ä–º–∏_ - —Ü–µ —Å—É–∫—É–ø–Ω—ñ—Å—Ç—å –ø—ñ–¥—Å–∏—Å—Ç–µ–º, —â–æ –∑–∞–±–µ–∑–ø–µ—á—É—é—Ç—å –∫–µ—Ä—É–≤–∞–Ω–Ω—è —Å—Ç–∞–Ω–æ–º —Ç–∞ –Ω–∞–ª–∞—à—Ç—É–≤–∞–Ω–Ω—è–º–∏ –µ–∫–∑–µ–º–ø–ª—è—Ä–∞ _–ü–ª–∞—Ç—Ñ–æ—Ä–º–∏_ —Ç–∞ —Ä–æ–∑–≥–æ—Ä–Ω—É—Ç–∏—Ö –Ω–∞ –Ω—ñ–π —Ä–µ—î—Å—Ç—Ä—ñ–≤ –∑–∞ –¥–æ–ø–æ–º–æ–≥–æ—é _GitOps_-–ø—ñ–¥—Ö–æ–¥—É —à–ª—è—Ö–æ–º –≤–Ω–µ—Å–µ–Ω–Ω—è –∑–º—ñ–Ω —É –≤—ñ–¥–ø–æ–≤—ñ–¥–Ω—ñ —Ä–µ–ø–æ–∑–∏—Ç–æ—Ä—ñ—ó —Å–∏—Å—Ç–µ–º–∏ –∫–æ–Ω—Ç—Ä–æ–ª—è –≤–µ—Ä—Å—ñ–π —Ç–∞ –∞–≤—Ç–æ–º–∞—Ç–∏—á–Ω–æ–≥–æ —ó—Ö –∑–∞—Å—Ç–æ—Å—É–≤–∞–Ω–Ω—è.
_The administrative zone of the Platform_ is a set of subsystems that provide management of the state and configurations of the Platform instance and the registries deployed on it using the GitOps approach, by making changes to corresponding repositories in the version control system and applying them automatically.

//_–ü—ñ–¥—Å–∏—Å—Ç–µ–º–∞ —É–ø—Ä–∞–≤–ª—ñ–Ω–Ω—è –ü–ª–∞—Ç—Ñ–æ—Ä–º–æ—é —Ç–∞ –†–µ—î—Å—Ç—Ä–∞–º–∏_ –∞–¥–º—ñ–Ω—ñ—Å—Ç—Ä–∞—Ç–∏–≤–Ω–æ—ó –∑–æ–Ω–∏ –Ω–∞–¥–∞—î –≤–µ–±-—ñ–Ω—Ç–µ—Ä—Ñ–µ–π—Å–∏ –¥–ª—è –∞–¥–º—ñ–Ω—ñ—Å—Ç—Ä–∞—Ç–æ—Ä—ñ–≤, –∑–∞ –¥–æ–ø–æ–º–æ–≥–æ—é —è–∫–∏—Ö –º–æ–∂–Ω–∞ –∫–µ—Ä—É–≤–∞—Ç–∏ –æ–Ω–æ–≤–ª–µ–Ω–Ω—è–º, –∫–æ–Ω—Ñ—ñ–≥—É—Ä–∞—Ü—ñ—î—é, –º–∞—Å—à—Ç–∞–±—É–≤–∞–Ω–Ω—è–º, —Ä–µ–∑–µ—Ä–≤–Ω–∏–º –∫–æ–ø—ñ—é–≤–∞–Ω–Ω—è–º, –≤—ñ–¥–Ω–æ–≤–ª–µ–Ω–Ω—è–º —Ç–∞ —ñ–Ω—à–∏–º–∏ —Å–ª—É–∂–±–æ–≤–∏–º–∏ —Ñ—É–Ω–∫—Ü—ñ—è–º–∏ _–ü–ª–∞—Ç—Ñ–æ—Ä–º–∏_.
_The Platform and registries management subsystem_ in the administrative zone offers web interfaces for administrators to manage updates, configurations, scalability, backup, restoration, and other service functions of the Platform.
