== Огляд

У цьому документі розглядаються функції, які можна використовувати для налаштування відображення вмісту на формах бізнес-процесів відповідно до локалі користувача або системних налаштувань реєстру. Ці функції дозволяють динамічно змінювати інтерфейс користувача на основі обраної мови, що покращує користувацький досвід.

Функції, описані в цьому документі, використовуються для налаштування умовного відображення компонентів на формах бізнес-процесів. Хоча вирази цих функцій нагадують JavaScript, фактично вони не є стандартним JavaScript-кодом. Замість цього, це умовні вирази, які інтерпретуються в контексті специфічного середовища платформи, такого як конструктор UI-форм або BPM-система.

Ці вирази застосовуються для визначення умов видимості або налаштування локалізації, і платформа обробляє їх у своєму внутрішньому середовищі. Отже, функції працюють у межах обмеженого синтаксису, підтримуваного платформою, але їх можна застосовувати для налаштування інтерфейсу без написання повноцінного JavaScript-коду.

== Функції для роботи з локалізацією

При моделюванні UI-форм, доступні наступні функції для налаштування локалізації:

* `getUserLanguage()` — визначає поточну мову, обрану користувачем, використовуючи значення з cookie `"Language"`. Повертає мовний код (наприклад, `"uk"` для української або `"en"` для англійської), який можна використовувати для умовного відображення контенту на формі, адаптуючи інтерфейс до мовних уподобань користувача.
+
При моделюванні UI-форми Платформа надає можливість отримати поточну або обрану локаль користувача, який працює над задачею, для використання в умовах відображення (Search Condition, SC), скриптах та інших налаштуваннях форми. Якщо користувач не обрав мову, використовується локаль реєстру за замовчуванням.
+
Ця функція забезпечує динамічне відображення інтерфейсу та контенту відповідно до мовних уподобань користувача, дозволяючи UI-формам автоматично підлаштовуватись під обрану локаль для покращення користувацького досвіду.

* `getRegistryLanguage()` -- визначає поточну мову, яка використовується в реєстрі або модулі, базуючись на значенні `"global.language"`. Ця функція використовується для встановлення стандартної мови інтерфейсу.

* `getRegistrySupportedLanguages()` -- повертає список мов, доступних у реєстрі, використовуючи значення `"global.supportedLanguages"`. Наприклад, `[ "uk", "en", "it" ]`.

Ці функції доступні для використання в усіх вкладках UI-форм, де можна вставляти JavaScript-вирази.

[IMPORTANT]
Значення мовного коду має відповідати стандарту *ISO 639-1*, наприклад, `en` для англійської, `uk` для української тощо.

== Приклад налаштування мовного контенту на формі

Нижче наведено приклад, як за допомогою цих функцій можна налаштувати відображення повідомлення мовою, обраною користувачем або налаштованою у реєстрі.

[TIP]
В інтерфейсі моделювання UI-форм редактор JavaScript-коду розширено підказками з використання службових функцій по роботі з мовами.

=== Створення нової форми

. Відкрийте
include::platform:ROOT:partial$templates/links/registry/administrative/admin-portal.adoc[]
. Створіть нову або відкрийте наявну
xref:registry-admin/admin-portal/version-control/candidate/create-new-change-request.adoc[версію-кандидат].
. У розділі *UI форми* натисніть кнопку btn:[Створити нову форму].
+
image::bp-modeling/forms/functions/form-functions-1.png[]

. Заповніть поля для бізнес- та службової назви форми:

* *Бізнес-назва* -- коротка описова назва форми.
* *Службова назва* -- внутрішня назва форми для використання у системі.

+
image::bp-modeling/forms/functions/form-functions-2.png[]

. Перейдіть на вкладку *Конструктор*. У меню ліворуч оберіть компонент *Content* та перетягніть його до панелі моделювання. У розділі *Display* введіть текст повідомлення для української локалі.
+
image::bp-modeling/forms/functions/form-functions-3.png[]
+
TIP: Детальніше про компонент *Content* ви можете дізнатися на сторінці xref:bp-modeling/forms/components/content.adoc[].

=== Налаштування умовного відображення за допомогою функцій локалізації

. На вкладці *Conditional* розгорніть розділ *Advanced Conditions*.
+
image::bp-modeling/forms/functions/form-functions-4.png[]

. У розділі *JavaScript* введіть умову для відображення компонента українською мовою, використовуючи функцію `getUserLanguage()`:
+
.Отримання локалі українською мовою
[source,javascript]
----
show = getUserLanguage() === "uk";
----
+
image::bp-modeling/forms/functions/form-functions-5.png[]

. Збережіть форму, натиснувши кнопку btn:[Save].

. Після збереження натисніть btn:[Створити форму].
+
image::bp-modeling/forms/functions/form-functions-6.png[]

=== Підсумок

Після налаштування умовного відображення контенту на основі локалі користувача або налаштувань реєстру, при запуску бізнес-процесу повідомлення на формі відображатиметься тією ж мовою, яку користувач обрав або яка встановлена як стандартна в реєстрі. Це дозволяє забезпечити зручний і зрозумілий інтерфейс, адаптований під індивідуальні потреби користувачів.
