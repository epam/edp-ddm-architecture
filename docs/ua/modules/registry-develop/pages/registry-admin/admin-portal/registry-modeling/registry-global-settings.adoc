Платформа надає можливість керувати глобальними налаштуваннями реєстру в інтерфейсі порталу адміністратора регламенту.

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-1.png[]

Наразі адміністратор регламенту може налаштувати такі параметри: ::

* xref:#support-email[Поштова адреса служби підтримки]
* xref:#support-channel-url[Посилання на канал зв'язку зі службою підтримки]
* xref:#registry-full-name[Повна назва реєстру]
* xref:#registry-short-name[Скорочена назва реєстру]
* xref:#ui-theme[Тема інтерфейсу]

Надалі перелік налаштувань буде розширено.

CAUTION: Редагування складових регламенту реєстру можливо внести лише в рамках версій-кандидатів на внесення змін. Для мастер-версії доступна лише опція перегляду.

IMPORTANT: Для того, щоб застосувати бажані параметри, натисніть кнопку `Зберегти зміни`, як показано на зображенні вище.

== Служба підтримки

[#support-email]
=== Поштова адреса служби підтримки

Якщо користувач Кабінету посадової особи, або отримувача послуг отримає критичну помилку, він зможе звернутись до підтримки за цією адресою.

Параметр задається у полі *Поштова адреса служби підтримки*.

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-2.png[]

[IMPORTANT]
====
Електронна адреса має бути у форматі `test@example.com`. Якщо введена адреса не відповідає заданому формату, користувач отримає відповідну валідаційну помилку:

`❗ Електронна адреса має бути в форматі "test@example.com"`

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-3.png[]
====

[WARNING]
====
Користувач не зможе вносити в налаштуваннях адреси доменів, що заборонені чинним законодавством України (наприклад, домени `mail.ru`, або `yandex.ru` тощо). Система видасть відповідну валідаційну помилку:

`❗ Дана поштова адреса не може використовуватись через політику безпеки`

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-4.png[]
====

[#support-channel-url]
=== Посилання на канал зв'язку зі службою підтримки

Поле *Посилання на канал зв'язку зі службою підтримки* надає можливість налаштувати та відображати відповідне посилання у Кабінетах надавача та отримувача послуг, що дозволить швидко отримувати допомогу та оперативно розв'язувати проблеми.

- Поле приймає лише валідні значення типу `URL`, з обов'язковим `://`. Такий патерн надає варіативність у використанні URL різного типу. Наприклад, `https://support.example.com` або `tg://channel.example.name` тощо.
+
image:registry-admin/admin-portal/global-settings/support-channel-url/support-channel-url-1.png[]

- При порушенні патерна спрацьовує валідація, яка забезпечує коректність введення даних.
+
image:registry-admin/admin-portal/global-settings/support-channel-url/support-channel-url-2.png[width=75%]

- У файлі конфігурації `values.yaml` використовується параметр `supportChannelUrl`. Значення цього параметра зміниться після xref:#submit-changes[].

*Відображення посилання на службу підтримки в кабінетах надавача та отримувача послуг:*

- Налаштоване посилання відображається у header-і на всіх сторінках Кабінетів: стартовій сторінці, сторінці автентифікації Keycloak, внутрішніх сторінках кабінету, формах задач, профілі тощо.
- Посилання клікабельне, система перенаправляє користувача на вказаний ресурс після натискання.
+
image:registry-admin/admin-portal/global-settings/support-channel-url/support-channel-url-3.png[]

== Назва реєстру

Розділ дозволяє вам налаштувати назви для вашого реєстру: _повну_ та _скорочену_.

[#registry-full-name]
=== Повна назва реєстру

_Повна назва_ -- це офіційна юридична назва реєстру. Використовується в офіційному листуванні, у документах, колонтитулах кабінетів тощо.

CAUTION: Може мати від 3 до 512 символів. Допустимі символи: `"А-Я"`, `"а-я"` української абетки, `"0-9"`, `"-"`, `","`, `"."`, апостроф (`'`), пробіл.

.*_Приклад. Повна назва реєстру ЄДР_*
``
ЄДИНИЙ ДЕРЖАВНИЙ РЕЄСТР ЮРИДИЧНИХ ОСІБ, ФІЗИЧНИХ ОСІБ - ПІДПРИЄМЦІВ ТА ГРОМАДСЬКИХ ФОРМУВАНЬ`
``

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-5.png[width=80%]

[#registry-short-name]
=== Скорочена назва реєстру

_Скорочену назву_ використовують у неофіційному спілкуванні, а також там, де недостатньо місця для повної назви, наприклад, у верхній частині кабінетів.

CAUTION: Може мати від 1 до 42 символів. Допустимі символи: `"А-Я"`, `"а-я"` української абетки, `"0-9"`, `"-"`, `","`, `"."`, апостроф (`'`), пробіл.

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-6.png[width=80%]

[#ui-theme]
== Тема інтерфейсу

Ця секція дозволяє встановити тему для інтерфейсів користувача.

image:registry-admin/admin-portal/global-settings/registry-global-settings-ua-7.png[width=75%]

Обрана тема застосовується для всіх Кабінетів на Платформі:

* Кабінет користувача/надавача послуг;
* Кабінет отримувача послуг;
* Кабінет адміністратора регламентів.

NOTE: Зміна теми впливає на інтерфейс Control Plane.

*Наразі підтримуються 2 теми:*

* [*] *Opensource UI*
* [*] *Diia Light* (_за замовчуванням_)

[#submit-changes]
== Застосування налаштувань

. Збережіть налаштування та застосуйте зміни до мастер-версії регламенту, використовуючи xref:registry-develop:registry-admin/admin-portal/version-control/candidate/overview-new-change-request.adoc#push-changes-master[відповідну процедуру].

. Після проходження пайплайну публікації регламенту, *MASTER-Build-registry-regulations*, налаштування реєстру буде змінено.
+
image::registry-admin/admin-portal/global-settings/registry-global-settings-ua-01.png[]

Наприклад, якщо ви змінили тему інтерфейсів, можна легко перевірити, чи спрацювала ваша конфігурація. Після виконання пайплайну, просто оновіть сторінку Адміністративного або іншого порталу.
