Функція вибору мови дозволяє користувачам вебпорталів реєстру динамічно обирати мову інтерфейсу через селектор, розміщений у верхній частині вебінтерфейсу. Ця функція доступна в таких інтерфейсах:

* Кабінет надавача послуг
* Кабінет отримувача послуг
* Кабінет адміністратора регламенту

Усі доступні мови формуються на основі налаштування *`global.supportedLanguages`* конфігурації реєстру.

== Доступність та налаштування мов

Селектор мови розміщено у верхній частині інтерфейсу клієнтських додатків. Він доступний на кожній сторінці й дозволяє обрати одну з підтримуваних мов. Перелік мов формується на основі налаштувань конфігурації реєстру, зокрема параметра `global.supportedLanguages`.

image:registry-admin/language-selector/admin-portal-language-selector-1.png[]

== Збереження вибраної мови

Вибрана користувачем мова зберігається у cookie з назвою `Language`, яка автоматично оновлюється при зміні мови в селекторі. Обрана мова застосовується в реальному часі для оновлення інтерфейсу.

== Автоматичне визначення мови при завантаженні додатка

При завантаженні додатка система перевіряє наявність і значення cookie `Language`:

* Якщо cookie існує і її значення відповідає одній із підтримуваних мов, ця мова автоматично обирається в селекторі й застосовується для інтерфейсу.
* Якщо cookie відсутня або її значення не відповідає підтримуваній мові, система створить нову cookie `Language` зі значенням мови за замовчуванням (визначеним параметром `global.language`) і застосує цю мову.

== Застосування мови в реальному часі

Зміни у виборі мови застосовуються негайно без необхідності перезавантаження сторінки. Це забезпечує безперервний користувацький досвід із моментальним оновленням мови інтерфейсу.

image:registry-admin/language-selector/admin-portal-language-selector-2.png[]

== Формування HTTP-запитів

Під час формування запитів на бекенд-сервіси, клієнтський додаток додає до кожного запиту HTTP-заголовок `Accept-Language` зі значенням поточної мови користувача.

== Локалізація повідомлень про помилки

Сервіс `form-submission-validation` обробляє заголовок `Accept-Language` і використовує його значення для локалізації повідомлень про помилки під час валідації форм.

== Збереження вибору мови для майбутніх сесій

Вибрана мова зберігається в cookie `Language`. При наступному вході в систему інтерфейс автоматично відображатиметься мовою, обраною раніше.
