== Загальний опис

*_Перегляд історії даних_* -- це механізм для відстеження змін у даних і їхнього походження, корисний для виявлення помилок або порушень цілісності.

Функція витягу історичних даних створює звіт про зміни в конкретному записі бази даних. Витяг генерується за шаблоном, який містить системні метадані та бізнес-дані таблиці. Бізнес-поля зберігаються у витягу в оригінальному вигляді, латиницею, і не належать до полів шаблону (наприклад, `user_name`, `birthdate`).

== Налаштування регламенту

Для формування витягу потрібно налаштувати шаблон у системі. Кожен шаблон повинен зберігатися у відповідній директорії та включати файли для різних мов.

=== Налаштування шаблонів витягів

Розміщення шаблонів має відповідати наступній структурі директорій залежно від налаштованого регіону обслуговування:

.Структура регламенту для шаблону `-uk` за замовчуванням та локалізованого шаблону `-en`
[plantuml]
----
@startsalt
{
{T
+ <&folder> **excerpts**
++ <&folder> **HistoryExcerpt**
+++ <&folder> css
++++ <&file> style.css
+++ <&folder> images
++++ <&file> global.png
+++ <&file> index.html.ftl
++ <&folder> **HistoryExcerpt-en**
+++ <&folder> css
++++ <&file> style.css
+++ <&folder> images
++++ <&file> global.png
+++ <&file> index.html.ftl
}
}
@endsalt
----

[IMPORTANT]
Значення мови має відповідати стандарту *ISO 639-1*. Наприклад, `en` для англійської, `fr` для французької тощо.

=== Правила вибору шаблону

1. Якщо витяг формується для реєстру в регіоні обслуговування Україна, система використає шаблони під директорією `*_HistoryExcerpt_*`. Мова налаштовується глобально у реєстрі: menu:Control Plane[Реєстри > Редагування реєстру > Локалізація].
2. Якщо мова, зазначена у запиті, не відповідає налаштованій мові реєстру, буде використано шаблон за замовчуванням (наприклад, `*_HistoryExcerpt_*` для України або `*_HistoryExcerpt-en_*` для глобального регіону).
3. Якщо мова реєстру змінена на _англійську_, і в регламенті є директорія `*_HistoryExcerpt-en_*`, буде використаний цей шаблон.

=== Опис полів шаблону

Шаблон витягу містить наступні поля:

[cols="1,2", options="header"]
|===
| Поле                    | Опис
| Час створення           | Час виконання операції, формат `dd.MM.yyyy HH:mm:ss`.
| Ідентифікатор автора    | Ідентифікаційний код особи, токен якої використовується для підпису даних у бізнес-процесі. Наприклад:
`xref:bp-modeling/bp/juel-functions/users/initiator.adoc[initiator]`,
`xref:bp-modeling/bp/juel-functions/users/system-user.adoc[system_user]`,
`xref:bp-modeling/bp/juel-functions/users/completer.adoc[completer]`.
| Код операції            | Тип операції: *створення*, *зміна* або *видалення*.
| Назва системи           | *_Службове поле_*. Назва підсистеми, де була виконана операція.
| Назва додатка           | *_Службове поле_*. Назва сервісу, який виконував операцію. Наприклад, `bpms`.
| Ідентифікатор бізнес-процесу | Унікальний ідентифікатор бізнес-процесу.
| Назва кроку в бізнес-процесі | Ідентифікатор завдання або активності у бізнес-процесі.
| Повне ім'я підписанта        | Повне ім'я підписанта або виконавця.
| Ідентифікаційний код ДРФО    | *_Службове поле лише для України_*. Ідентифікаційний код фізичних осіб або ФОП в Україні. Підтягується з КЕП.
| Код ЄДРПОУ                   | *_Службове поле лише для України_*. Код юридичних осіб або організацій в Україні. Підтягується з КЕП.
|===

=== Змінні бізнес-поля, присутні у витягу

Бізнес-поля витягу можуть відрізнятися для кожної таблиці й будуть специфічними для кожної сутності. Їхнє відображення у витягу відбувається без змін, незалежно від налаштувань мови шаблону (`-uk`, `-en` тощо).

== Процес генерування витягу на основі шаблону

Згенерувати витяг історичних даних можна наступним чином:

. Увійдіть в консоль *Openshift*.
. Оберіть проєкт із вашим реєстром. Наприклад, `demo-registry`.
. В меню ліворуч відкрийте розділ *Pipelines* та знайдіть пайплайн *history-excerptor*.
+
image:registry-admin/excerpts/history-excerptor/history-excerptor-1.png[]

. Відкрийте вкладку *Parameters* та натисніть menu:Actions[`Start`].
+
image:registry-admin/excerpts/history-excerptor/history-excerptor-2.png[]

. У вікні menu:Start Pipeline[Parameters] заповніть наступні поля:
* *_Обов'язкове_* `table_name` -- назва таблиці, з якої потрібно виконати витяг. Наприклад, `user`.
* *_Обов'язкове_* `uuid` -- ідентифікатор запису в таблиці, наприклад, UUID користувача.
* *_Опціональне_* `log_level` -- рівень логування: `info` або `debug`.
. Натисніть btn:[Start].

+
image:registry-admin/excerpts/history-excerptor/history-excerptor-3.png[]

== Отримання згенерованого витягу

. Відкрийте menu:Pipelines[Tasks > TaskRuns] та знайдіть задачу із назвою `*history-excerptor-run*-<ідентифікатор-витягу>`.
+
image:registry-admin/excerpts/history-excerptor/history-excerptor-4.png[]

. Перейдіть у меню menu:TaskRun details[Logs].
. Внизу логу знайдіть HTTPS-посилання на згенерований витяг у форматі `.pdf`.
+
image:registry-admin/excerpts/history-excerptor/history-excerptor-5.png[]

. Скопіюйте посилання та вставте у браузері, щоб завантажити PDF-файл з історією змін.
