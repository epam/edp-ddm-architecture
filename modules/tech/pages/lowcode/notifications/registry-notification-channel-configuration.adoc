== Налаштування каналів зв'язку для відправки повідомлень користувачам реєстру

=== Інтерфейс управління платформою
Налаштування каналів зв'язку окремого реєстру реалізовано за допомогою розширення _Адміністративного інтерфейсу управління платформою_ (*control-plane*) на етапах створення та редагування реєстру.

Наразі, зміна налаштувань реєстру можлива лише користувачам, яким призначено одну з системних ролей:

- _Адміністратор платформи_ (*cp-cluster-mgmt-admin*) - на етапі створення реєстру / зміни налаштувань існуючого реєстру
- _Адміністратор реєстру_ (*cp-registry-admin-<registry>*) - зміна налаштувань існуючого реєстру

TIP: З детальною інформацією щодо створення реєстрів можна ознайомитись у розділі xref:platform:admin:registry-management/control-plane-create-registry.adoc[Створення реєстру].

Наразі платформа підтримує відправлення електронних повідомлень з використанням _SMTP_ у якості протоколу комунікації та однієї з наступних опцій налаштувань поштового сервера в залежності від вимог реєстру:

- _Платформенний поштовий сервер_ (*platform-mail-server*) - Поштовий сервер, який розповсюджується у якості платформенного сервісу та доступний для використання усіма реестрами одного екземпляру платформи
- _Зовнішній поштовий сервер_ (*external-mail-server*) - Зовнішній відносно платформи поштовий сервіс (_Gmail, тощо._)

=== Канонічний вигляд налаштувань каналів зв'язку реєстру

.Канонічний вигляд структури *OpenShift ConfigMap* "notification-service-channel-configuration"
[source, yaml]
----
kind: ConfigMap
apiVersion: v1
metadata:
  name: notification-service-channel-configuration
data:
  application.yaml: |-
    notifications:
      email:
        host: smtp.gmail.com
        port: 587
        properties:
          mail:
            transport:
              protocol: smtp
            smtp:
              auth: true
              starttls:
                enable: true
      diia: ...
      messenger: ...

----

.Канонічний вигляд структури *OpenShift Secret* "notification-service-channel-configuration"
[source, yaml]
----
kind: Secret
apiVersion: v1
metadata:
  name: notification-service-channel-configuration
data:
  notifications.email.username: <username>
  notifications.email.password: <password>
----
